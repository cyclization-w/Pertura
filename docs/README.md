## Product model

```text
pertura_core
  contracts + scope + promotion policy
        |
pertura_workflow
  capabilities + planner + dependency resolution
        |
pertura_runtime
  authority sessions + five tools + CLI/dashboard

pertura_bench -> exercises the complete product path
agent_bundle  -> shared provider-neutral operational and biological skills
adapters      -> Claude runtime plus import-safe future OpenAI contract
pertura_gate  -> legacy compatibility and regression only
```

The five bundled skills influence agent behavior only; their hash is provenance, not scientific authority. The Claude adapter is runnable, while the OpenAI adapter is an import-safe schema skeleton and reports not ready. Exploratory capability results may be committed to the dependency DAG, but they have no trusted receipt and cannot support a strong measured statement. Synthetic fixtures establish code readiness only; they never change capability trust.

## Release boundary

The expected local checkpoint is:

```text
build_version: 0.2.0a9
repository_ready: true
runtime_spine_ready: true
code_ready: true
local_fixture_ready: true
real_benchmark_ready: false
release_ready: false
```

Optional environment readiness is reported from the current machine. Real-data locks/verdicts and expert-adjudicated CRISPRi/CRISPRa profiles remain external release blockers.
