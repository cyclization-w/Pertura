paper-benchmark checkpoint. Historical evidence-lattice, stage, smoke, and
paper-facing materials are isolated under `legacy/` and are not current product
architecture.

## Current product model

```text
pertura_core
  frozen contracts + scope + promotion
        |
pertura_workflow
  capabilities + planner + dependency resolution + runners
        |
pertura_runtime
  projects + assets + authority sessions + turns + five tools + adapters

pertura_bench -> scientific and agent evaluation through the product path
agent_bundle  -> provider-neutral operational and biological skills
legacy/       -> excluded evidence/registrar/stage archive and separate tests
```

There is one active authority spine:

```text
ResultEnvelope -> pertura_core.promotion -> TurnFinal / report
```

`phase` is presentation metadata, not scientific authority. Skills influence
behavior but never receipts, dependencies, or promotion. Exploratory results
may enter the DAG as `validated_untrusted`, but they cannot support a strong
measured statement.
