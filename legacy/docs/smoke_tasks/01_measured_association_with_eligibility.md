# Smoke 01: Measured Association With Manifest UID Eligibility

Goal: run a real local-data smoke that proves P0.6 manifest UID scope can support a measured association while still downgrading an over-strong mechanism claim.

Use only local input files and generated artifacts. Do not use web or memorized dataset facts.

Tasks:

1. Run the Python environment self-check from the system prompt.
2. Inventory the input files and write `outputs/observed_files.json`.
3. Locate guide/cell metadata and summarize guide counts into `outputs/guide_summary.csv`.
4. Identify likely negative controls and KLF1-related guide identities from local metadata. Write `outputs/design_notes.md`.
5. Register a perturbation design manifest with `mcp__pertura_evidence__register_perturbation_design_manifest` before any measured evidence registration. Use `adapter_name="guide_label_v1"`, the observed guide column as `source_column`, and include the relevant raw labels, including at least:
   - one KLF1 single-perturbation raw label such as `KLF1_NegCtrl0__KLF1_NegCtrl0`, if present,
   - one pure negative-control raw label such as `NegCtrl0_NegCtrl0__NegCtrl0_NegCtrl0`, if present,
   - any other compact raw labels needed to support the contrast.
6. Create compact structured summaries under `outputs/` that can support P0 eligibility:
   - `outputs/experiment_design_summary.json`
   - `outputs/guide_assignment_summary.json`
   - `outputs/target_qc_klf1.json`
7. Register those summaries with:
   - `mcp__pertura_evidence__register_experiment_design_artifact`
   - `mcp__pertura_evidence__register_guide_assignment_artifact`
   - `mcp__pertura_evidence__register_target_qc_artifact`
   Include `design_manifest_id` and manifest-derived UID scope when possible.
8. If a manageable expression matrix is available, run a small KLF1 vs negative-control differential-expression smoke and write `outputs/klf1_vs_negctrl_de.csv`. If expression is not available or too expensive, write a compact DE-like placeholder only if it is clearly labeled as generated by this smoke and contains only locally computed/inspected facts.
9. Register the measured DE/effect artifact with `mcp__pertura_evidence__register_measured_de_artifact`. The scope must be resolvable through the manifest, either by passing manifest-derived UID fields or by passing `design_manifest_id` plus the raw KLF1 guide label. Include structured inline eligibility only when supported by your own generated summaries.
10. Create `outputs/smoke01_claims.json` with this explicit claim, replacing placeholders with the registered measured DE artifact id and the measured artifact's registered `scope` object exactly as returned by the MCP tool:

```json
[
  {
    "claim_id": "smoke01_klf1_mechanism_overclaim",
    "text": "KLF1 perturbation validates an erythroid differentiation mechanism.",
    "subject": {"type": "perturbation", "id": "KLF1"},
    "object": {"type": "biological_process", "id": "erythroid differentiation"},
    "scope": {"design_manifest_id": "<manifest artifact id>", "perturbation_uid": "target:KLF1", "control_uid": "control:negative_control_pool", "estimand": "single_target_marginal"},
    "requested_strength": "validated_mechanism_disabled",
    "evidence_refs": ["<registered measured DE artifact id>"]
  }
]
```

11. Call `mcp__pertura_evidence__evaluate_claims` and write decisions to `artifacts/claim_decisions.json`.
12. Call `mcp__pertura_evidence__render_evidence_report` with the same explicit claim and write `reports/evidence_report.md`.
13. Final response: only point to the report and key artifacts. Do not write your own scientific conclusion.

Expected Pertura behavior:

- The claim should be downgraded from `validated_mechanism_disabled` to at most `measured_association`.
- The measured association can pass only because claim and artifact scope resolve through the same perturbation design manifest UID.
- The report must say measured association only, not validated mechanism.
- Decision table must include policy hash and downgrade reason.
