# GitHub Repository Manifest

This directory is a clean export intended to become the public/private GitHub repository root.

## Included

- `src/pertura_gate/`: trusted evidence lattice, identity, policy, resolver, and renderer code.
- `src/pertura_runtime/`: Claude runtime adapter, finalizer, run manifests, and evidence MCP tool surface.
- `src/pertura_bench/`: P0.7 strong-baseline harness and benchmark-only surface evaluator.
- `tests/`: focused gate/runtime/bench unit and integration tests.
- `scripts/`: top-level smoke/benchmark helper scripts only.
- `docs/`: clean architecture docs, smoke task prompts, P0.7 prompts, skill cards.
- `pyproject.toml`: build metadata and dependency extras.
- `README.md`: GitHub-facing repository entry point.
- `.gitignore`: excludes local runs, generated outputs, datasets, caches, and secrets.

## Excluded

- `.git/`
- `.claude_runs/`
- `.p07_runs/`
- `.smoke_runs/`
- `.pertura_workspace/`
- `.pytest_cache*`
- `.tmp/`
- top-level `outputs/`, `reports/`, `artifacts/`
- `scripts/.smoke_runs/`
- all `__pycache__/` and `*.pyc`
- `generated_by_gpt.txt`
- local datasets and API keys
- legacy general-purpose platform modules that are not part of Pertura's gate/runtime/bench trust split

## Rationale

The exported repo keeps Pertura's reproducible source, tests, architecture docs, smoke prompts, and benchmark harness material while excluding run-local evidence outputs and private/local execution state. Runtime outputs are generated by users locally and should not be committed by default.

The source tree is intentionally split by trust domain:

```text
pertura_gate      trusted deterministic gate core
pertura_runtime   untrusted Claude/agent runtime adapter
pertura_bench     benchmark evaluator and strong-baseline harness
```

`pertura_gate` must not depend on `pertura_runtime` or `pertura_bench`; this is enforced by tests.
