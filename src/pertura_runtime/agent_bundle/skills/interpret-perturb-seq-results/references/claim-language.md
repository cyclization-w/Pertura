# Claim Language

Use wording matched to the committed source and status.

- Observed metadata: "The recorded design identifies..." or "The dataset contains..."
- Trusted measured result: "Within the tested scope, the analysis estimated..." Include direction, uncertainty, and limitation.
- Validated untrusted candidate: "An exploratory capability estimated..." or "The candidate analysis suggests..."
- Derived summary: "A summary derived from the committed effect matrix shows..."
- Prediction: "The model predicts..." State split and evaluation scope.
- Curated prior: "Prior knowledge reports..." Keep it separate from the current measurement.
- Hypothesis: "One hypothesis is..." State an alternative and a discriminating experiment.

Avoid "proves," "validates a mechanism," "causes fate conversion," or "scientifically certified" unless a specific promoted statement and evidence scope permit that wording. Do not use a diagnostic `screen_passed` status as universal scientific validation.
